7. Layout Notes

This section provides general guidelines and considerations for the application layout, focusing on a mobile-first approach.

Core Principles

  • Mobile-First: Design components and layouts to be optimal on small screens first. Use Tailwind's responsive prefixes (sm:, md:, lg:, xl:, 2xl:) to adapt and enhance layouts for larger screens.
  • Consistency: Strive for consistent use of spacing, padding, and margins using Tailwind's spacing scale. This creates visual harmony.
  • Clear Hierarchy: Use typography, color, and shadows (as defined in this system) to establish a clear visual hierarchy, guiding the user's attention to important elements.
  • Accessibility: Ensure sufficient color contrast, keyboard navigability, and semantic HTML for all interactive elements.

Key Screens & Structure

  • Onboarding: Likely a series of full-screen views on mobile, guiding the user step-by-step. Desktop could use a modal or a focused centered layout.
  • Main Dashboard:
    • Mobile: Vertical scroll is key. The circular timer should be prominent at the top, followed by meal listings. "Eyebrow" action buttons might be fixed or appear contextually.
    • Desktop: More horizontal space allows for a potential multi-column layout, perhaps with the timer and key stats in one column, and meals in another.
  • Calendar Dashboard:
    • Mobile: A compact monthly view, with tappable days to show meal summaries or navigate to a daily detail view.
    • Desktop: A larger, more detailed calendar view.
  • Current Meal Dashboard / Recipe View:
    • Mobile: Clear, sequential presentation of ingredients, instructions, and nutrition. Tabs or accordions might be used for sections.
    • Desktop: Could use a two-column layout (e.g., ingredients/nutrition on one side, steps on the other).
  • Shopping List: A clear, checklist-style layout, potentially grouped by store or aisle.
  • Settings: Typically a list-based navigation to various setting sub-sections.

Navigation

  • Mobile Primary Navigation: A bottom tab bar is recommended for frequently accessed sections (e.g., Dashboard, Calendar, Shopping, Settings). Icons should be clear and accompanied by text labels for accessibility.

  • Desktop Primary Navigation: A collapsible sidebar could be suitable if the application has many top-level sections. For simpler apps, a top navigation bar might suffice.
  • Secondary Navigation: Within sections (e.g., Settings), use clear sub-headings or tabs.
  • Back Navigation: Consistent "Back" buttons or gestures are crucial, especially on mobile.

Grid and Spacing

Tailwind's default spacing scale (multiples of 0.25rem) should be used for padding, margins, and gaps to maintain visual rhythm. For page layouts, CSS Grid or Flexbox will be the primary tools.
