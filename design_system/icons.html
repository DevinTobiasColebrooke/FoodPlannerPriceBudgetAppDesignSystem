9. Icons

Icons are sourced from Flaticon and placed in the ./svgs/ directory. This page demonstrates referencing them via relative paths using tags. For icons needing CSS color control (e.g., single-color UI icons matching text color), inlining the SVG is preferred in component demos.

Important: When using , Tailwind text color classes (like text-fp-primary-green) will **not** change the SVG's internal colors. The icon will render with colors defined in the SVG file itself. Always check Flaticon licensing for commercial use and attribution requirements.

9.4. Using Icons for UI Elements (Inlining for CSS Color Control)

For UI icons where color needs to be controlled by CSS (e.g., matching text color, hover states), inlining the SVG is preferred. Here are examples of where you might do that elsewhere in the design system:

Close Icon (Inlined, text-fp-white)
Settings Icon (Inlined, text-fp-white)

To achieve the color control above (e.g., white icon on green background), the SVG would be inlined, and its fill or stroke attributes set to currentColor or specific paths targeted, with the parent element having the desired text-fp-white class.
